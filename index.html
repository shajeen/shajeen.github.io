std::optional

  • Is not going to dynamic allocation on it's own

  • optional set the value and call destructor when exit.

  • not supported by constexpr

example:

```c++ include

int main() {
std::optional o{13}; // if init return 13 return o.value_or(2); // else return 2 }

```

more example from cppreferance.com

```c++ #include
#include #include

std::optional create(bool b) { if (b) return "Godzilla"; return {}; }

auto create2(bool b) { return b ? std::optional{"Godzilla"} : std::nullopt; }

int main() { std::cout << "create(false) returned" << create(false).value_or("empty") << ''; if (auto str = create2(true)) { std::cout << "create2(true) returned " << *str << ''; } }


#### Output:

create(false) returned empty create2(true) returned Godzilla ```
